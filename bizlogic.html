查询下载更新

  1. 查询成功：
    检测是否有新版本，有新版本弹框提示更新，没有新版本，则判断是否是新用户
  2. 查询失败：
    判断是否是新用户
  3. 用户点击更新且 url 不为空，则开启下载apk线程，同时弹出下载进度框
    • 若外置储存卡存在则提示用户，同时停止下载；
    • 若外置储存卡存在则进行如下初始化工作；
      • 默认保存目录是外置储存卡下/Download/APK,若文件夹不存在则创建文件夹；
      • 默认文件名为截取 url 最后一个“/”后的字符串；
      • 默认会覆盖已有的同名文件；
    • 下载文件，同步更新进度条；
    • 下载完成，进度条更新到100%；
  4. 文件下载完成开始通知系统安装，同时杀死当前进程（关闭apk）

列表页预加载机制

  1. 变量A：定义顶部距离底部指定条目时开始预加载（默认主页10条，最新成交页7条，分类页10条）；
  2. 变量B：记录最后一次预加载时，顶部条目的索引；
  3. 监听列表页滚动事件，并实时判断，如果满足一下两个条件，则进行预加载，同时将更新最后一次预加载时顶部条目的索引：
    • 列表总条目减去当前顶部条目索引满足 A；
    • 当前顶部条目索引不等于 B；
  4. 在下拉刷新的时候将 B重置。

详情页

  1. 启动场景：
    • 只传入ProductId；
    • 只传入新用户的ProductId；
    • 传入PeriodId；
  2. 若只有ProductId，则默认通过ProductId查询最新期，若获取到最新期数列表为空，则判定为商品已下架。若最新期数列表不为空，则获取到最新一期的PeriodId；
  3. 通过 PeriodId 或者新用户的 ProductId 查询最新一期详情。1）若查询失败，则默认弹toast告知用户原因，未关闭 加载框，并终止后续操作；2）若查询成功，关闭加载框，并更新页面UI，如下：
    • 商品图片集
    • 专区角标图片
    • 市场价格
    • 商品标题
    • 可否差价购买
    • 竞拍细则（起拍价、加价幅度...）
    • 底部是否收藏
    • 底部每次竞拍的手续费
    • 底部默认出价
    • 判定当前一期状态
      • 判定本期为结束或异常
        • 停止轮询最新竞拍价任务/停止轮询竞拍助手任务
        • 设置倒计时为结束状态
        • 设置 水印 图，并停止图片集的滚动
        • 更新我的消耗
        • 更新底部视图（前往下期/返币/差价购等）
        • 若可分享，则添加分享按钮
        • 若本期异常，则弹框告知用户系统异常
      • 判定本期为首次倒计时或正在拍
        • 开启轮询最新竞拍价任务
  4. 在步骤3中更新视图的同时，并发执行请求本期晒单、往期成交、竞拍规则，等待数据全部获取到，按照如下优先级显示：
    • 若有晒单，优先显示晒单；
    • 若无晒单，且有往期成交，则显示往期成交；
    • 既无晒单，又无往期成交，则显示竞拍规则。
  5. 在步骤3中更新视图的同时，并发执行请求最近3条出价记录，若查询成功，则：
    • 显示最新出价记录
    • 判定当前一期状态为结束或异常
      • 更新当前竞拍价
      • 更新竞价状态说明（恭喜您/本次竞拍结束）
  6. 轮询最新竞拍价任务，默认500ms发起一次请求
